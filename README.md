# site-mk-pages

本项目用于归档和发布多个独立的 HTML 页面。仓库本身不针对任何特定域名或具体网站，仅作为静态页面文件的集合，便于统一管理和版本追踪。

## 项目简介

本仓库主要存放各类独立编写的 HTML 页面，这些页面彼此之间没有依赖关系，各自可以独立运行。仓库不涉及后端逻辑或数据库，所有内容均为前端静态文件。适合用于保存示例页面、临时页面或小型文档页面。

## 目录说明

```
site-mk-pages/
├── index.html          # 首页或导航页（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
└── assets/             # 公共资源（图片、样式、脚本等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：每个 HTML 文件对应一个独立页面，文件名应能反映内容用途。
- `assets/`：存放页面共享的静态资源，按类型分类。
- `index.html`：可选项，用于提供页面索引或入口。

## 页面归档说明

新增页面时，请遵循以下规则：

1. 页面文件统一放入 `pages/` 目录，避免在根目录堆积。
2. 页面内引用的资源建议使用相对路径，便于本地预览和部署。
3. 每个页面应自包含，不依赖其他页面的特定状态。
4. 如有公共样式或脚本，可放入 `assets/` 目录并在各页面中引用。

## 维护说明

- 本仓库不包含自动部署脚本，如需发布，请自行配置静态托管服务。
- 提交前请确认页面在主流浏览器中可正常显示。
- 不建议存放敏感信息或动态内容。
- 如需删除旧页面，请同时移除相关资源引用。

## 许可

本项目未指定特定许可证，代码仅作为示例或归档用途提供。如有需要，请自行添加许可证文件。
